## Affected Commands

- `botstack install` — primary interactive installer flow.
- `botstack install --plan` — dry-run flow that must make no writes.

## Key Interactions to Verify

- User selects install-all and reviews install directories before execution.
- User selects individual tools and only those tools appear in the plan.
- User selects multiple first-class harness targets and sees config files touched.
- User runs `--plan` and no files are written under the fake home directory.
- User reruns after a partial failure and sees honest prior state from the ledger.

## Edge Cases

- Unsupported OS.
- Missing package manager.
- Existing harness config with user content.
- Malformed harness config.
- Duplicate managed block.
- Backup directory permission denied.
- Tool install command timeout.
- Verification command fails after install command succeeds.
- Ledger write fails.

## Critical Paths

- Full install-all flow with mocked tool commands across fake macOS and Linux homes.
- Per-tool selection flow with one plugin selected.
- Config mutation flow with backup, typed merge, verification, and rollback.
- Failure recovery flow where one tool fails and the final report is still accurate.
- Docs generation flow where manifest install paths and verification commands appear in generated reference tables.
